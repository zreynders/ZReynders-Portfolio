Search Algorithms

Breadth-First Search

The breadth-first search algorithm expands the shallowest node in the frontier. This means that it will always find a path with the minimum points (not neccessarily the shortest distance though).

Depth-First Search

The depth-first search algorithm expands an entire branch and then back-tracks if it did not find the target node. This can be efficient in some trees, but not a graph. This algorithm is not guaranteed to return the shortest distance or minimum points.

Greedy Best-First Search

The greedy best-first search algorithm uses a defined heuristic to estimate the distance between the frontier points and the goal and chooses the best one. It does not consider distance travelled and so may not find the shortest path. (the heuristic being used here is euclidean distance between the point and the target)

Algorithm A

The Algorithm A algorithm uses a defined heuristic to estimate the distance between the frontier points and the goal along with the distance travelled and chooses the best one. Because it takes into account the distance travelled it will find the shortest path, but can be slower than the Greedy Best-First algorithm.
